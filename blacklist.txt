# 사용법
#   - 한 줄에 태그 하나. '#' 로 시작하는 줄은 주석(무시).
#   - WD tagger(app.py)는 '_' 를 공백으로 바꿔 출력하므로 여기서도 공백 표기.
#     (언더스코어 파이프라인이면 "artist name" -> "artist_name" 으로 치환해서 사용)
#
# 핵심 원칙 (Artist STYLE LoRA 기준)
#   1) 내용(content) 태그는 남긴다 → 포즈/의상/배경/표정/구도는 프롬프트로 제어해야
#      하므로 캡션에 있어야 한다. 지우면 그 요소가 트리거에 통째로 구워진다.
#   2) 식별/메타/소스 노이즈는 항상 제거 → 화풍과 무관하고 학습을 오염시킨다.
#   3) "화풍 그 자체" 태그(매체/셰이딩 등)는 데이터셋 전체에서 균일할 때만 제거 →
#      그래야 그 특징이 트리거로 흡수된다. 섞여 있으면 남겨서 제어권을 유지한다.
#      → 이런 항목은 아래에서 기본 주석 처리(#) 해뒀다. 균일하면 주석을 풀어라.
#
#   ※ Character(캐릭터) LoRA라면 반대로 hair color/eye color 같은 고유 외형도
#     제거해 트리거에 묶는다. 이 파일은 STYLE 용이라 외형은 남겨둔다.
# =============================================================================

#     · 의상/소품:  dress, hoodie, glasses, holding cup ...
#     · 배경:       white background, simple background, outdoors, cafe ...
#     · 표정:       smile, blush, open mouth ...
#     · 외형(STYLE 한정 유지): long hair, blonde hair, blue eyes ...
